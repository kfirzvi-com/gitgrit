    How to generate a token? Select a platform first

    URL must start with https:// (e.g. https://gitlab.example.com)

    Generate a GitHub Access Token

    1. Open Fine-grained token settings (github.com)
    2. Under Resource owner, select the organization whose repositories you want to connect
    3. Set Repository access to "All repositories" or select specific ones
    4. Under Permissions → Repository permissions, grant:
      • Contents — Read-only
      • Metadata — Read-only
      • Webhooks — Read and write
    5. Click Generate token and copy it

    Tip: Fine-grained tokens are scoped per organization. Create one token per organization you want to connect.

    Generate a GitLab Access Token

    1. Open gitlab.com and go to Avatar → Edit profile → Access → Personal access tokens
    2. From the dropdown, select Personal access token (legacy)
    3. Give it a descriptive name (e.g. "GitGrit") and set an expiration date
    4. Select the scope:
      • api — required for listing projects, reading repository data, and managing webhooks
    5. Click Create personal access token and copy it

    Tip: You can also use a group access token (Group → Settings → Access tokens) to scope access to a specific group's projects without tying it to a personal account.

    Note: GitLab's fine-grained tokens (Beta) don't yet support webhook management, which GitGrit requires for automatic policy evaluation. Use a legacy token for now.
